# tauri-plugin-ageverify

iOS-only Tauri plugin that exposes Apple's **Declared Age Range API** so the app
can hard-block users under 13 at launch (App Store age-assurance requirement).

## What it does

- `invoke('plugin:ageverify|request_age_range')` → `{ ageRange, available, declined }`
  - `ageRange`: `"under13" | "over13" | "unknown"` (we request a single 13-year age gate)
  - `available`: `false` on iOS < 26.2, non-iOS platforms, or on error
  - `declined`: the person (or their guardian) declined to share their age range

The frontend (`app/src/lib/ageverify.ts` → `app/src/contexts/AgeGate.tsx`) blocks
only on an explicit, available `under13`; everything else passes (fail-open).

## Privacy

The age band is used **only on-device** to decide whether to show the block
screen. It is never persisted raw nor transmitted anywhere — only a pass/fail
boolean (`ageverify:passed`) is cached in `localStorage`. This preserves the
app's "no personal information collection" stance.

## Requirements / build notes

- **iOS 26.2 SDK + Xcode 26.2.** The `DeclaredAgeRange` framework only exists in
  that SDK. The Swift code is guarded with `#if canImport(DeclaredAgeRange)` and
  `@available(iOS 26.2, *)`, so it still compiles on older SDKs (the runtime path
  simply reports `available: false`). The app's deployment target stays at 14.0.
- **Entitlement** `com.apple.developer.declared-age-range` is added in
  `src-tauri/gen/apple/project.yml` and `world-app_iOS/world-app_iOS.entitlements`.
  You must also enable this capability for the App ID in the Apple Developer
  portal and regenerate the provisioning profile (manual, out-of-repo step).
- `ios/.tauri/tauri-api` is generated by the Tauri CLI during `pnpm tauri ios
  build` (git-ignored), same as the other local plugins.

## ⚠️ Verify the Swift call against the 26.2 SDK

`ios/Sources/AgeVerifyPlugin.swift` calls
`AgeRangeService.shared.requestAgeRange(ageGates: 13, in:)` and switches on
`.declinedSharing` / `.sharing(range)`. The exact signature, response type, and
`range` shape should be confirmed against the shipping iOS 26.2 SDK
(https://developer.apple.com/documentation/declaredagerange/) and adjusted if
Apple's API differs.
